

这题本质考的是 **EIGRP 自动汇总（auto-summary）导致的非连续网络问题**。

### 为什么会丢包

R1 后面是 **172.16.1.0/24**
R3 后面是 **172.16.2.0/24**

它们中间经过的是 **10.1.x.x** 网段，所以这是一个典型的 **discontiguous network（非连续网络）** 场景。

如果开启了 EIGRP 自动汇总，R1 和 R3 在跨主类网络边界通告时，会把各自的 /24 路由汇总成：

**172.16.0.0/16**

于是 R2 会学到两条到 **172.16.0.0/16** 的路由，分别指向左右两个方向。这样去往某个具体地址，比如 **172.16.2.7** 的流量，可能被错误地转发到 R1 那边，结果就被丢掉了。

------

### 解决办法

要让 R2 学到的不是汇总路由，而是真实的子网：

- 172.16.1.0/24
- 172.16.2.0/24

这样它才能准确把发往 **172.16.2.7** 的流量送到 R3。

所以考试答案给 **B**：
**在 R2 上关闭 auto-summary**

------

### 为什么其他选项不对

**A. 插入 /24 floating static**

- 这是人为补丁，不是正确解决 EIGRP 汇总问题的标准方法。

**C. 在三台路由器上启用 auto summarization**

- 这会让问题更严重，不会解决。

**D. 插入 /16 floating static**

- 还是汇总路由，无法区分 172.16.1.0/24 和 172.16.2.0/24。

------

### 你做题时的秒杀判断

看到这几个特征就要立刻想到：

- **EIGRP**
- **172.16.x.x 子网分散在不同地方**
- **中间用 10.x.x.x 连接**
- **出现丢包 / 负载分担 / summary route**

=> 基本就是 **auto-summary 导致的 discontiguous network 问题**
=> 选 **disable auto-summary**

------

这个问题其实是 **EIGRP auto-summary 的历史设计问题**。我给你一步一步解释，你就会彻底理解为什么会出现这种奇怪的丢包。

------

# 1 先理解什么是 EIGRP auto-summary

EIGRP 在 **跨 major network（主类网络）时**会自动汇总。

比如：

| 子网          | 主类网络      |
| ------------- | ------------- |
| 172.16.1.0/24 | 172.16.0.0/16 |
| 172.16.2.0/24 | 172.16.0.0/16 |

如果开启 **auto-summary**，路由器在跨主类网络时只会通告：

```
172.16.0.0/16
```

而不是具体子网。

------

# 2 看这道题的拓扑

真实网络是：

```
172.16.1.0/24 ---- R1
                     \
                      R2
                     /
172.16.2.0/24 ---- R3
```

中间连接网络是：

```
10.1.1.0
10.1.2.0
```

注意：

**10.x.x.x 是另一个 major network**

所以 EIGRP 会触发 **auto-summary**。

------

# 3 auto-summary 会发生什么

R1 发送给 R2 的不是：

```
172.16.1.0/24
```

而是：

```
172.16.0.0/16
```

R3 发送给 R2 的也不是：

```
172.16.2.0/24
```

而是：

```
172.16.0.0/16
```

于是 R2 的 routing table 变成：

```
172.16.0.0/16 via R1
172.16.0.0/16 via R3
```

R2 **无法区分具体子网**。

------

# 4 为什么会丢包

假设 R4 要访问：

```
172.16.2.7
```

R4 → R2

R2 查 routing table：

```
172.16.0.0/16
```

但是有 **两个 equal-cost path**：

```
via R1
via R3
```

EIGRP 会 **load balancing**。

所以流量可能被送到：

```
R1
```

但 R1 只知道：

```
172.16.1.0/24
```

它 **没有 172.16.2.0/24**

于是：

```
packet drop
```

------

# 5 这就叫

**Discontiguous Network Problem**

意思是：

同一个 major network 被 **分割在不同位置**。

例如：

```
172.16.1.0/24   (左)
10.x.x.x
172.16.2.0/24   (右)
```

这就是 **非连续网络**。

------

# 6 解决方法

关闭 auto-summary。

```
router eigrp 100
 no auto-summary
```

这样 R2 收到的是：

```
172.16.1.0/24 via R1
172.16.2.0/24 via R3
```

路由表：

```
172.16.1.0/24 → R1
172.16.2.0/24 → R3
```

访问：

```
172.16.2.7
```

R2 就只会走 **R3**。

问题消失。

------

# 7 一句话总结

这题核心逻辑：

```
EIGRP auto-summary
+ discontiguous network
= summary route conflict
= packet drop
```

解决：

```
no auto-summary
```
