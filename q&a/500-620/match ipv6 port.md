匹配**协议端口**和校验**子网范围**。

#### **第一步：协议与端口号匹配**

这是最简单的切入点，你可以直接排除掉大部分干扰项：

- **NTP**: 端口号 **123**
- **Syslog**: 端口号 **514**
- **HTTP**: 端口号 **80**
- **HTTPS**: 端口号 **443**

#### **第二步：子网范围校验 (以 NTP 为例)**

我们来看为什么 **::1f** 属于 **::c/126**：

- **子网掩码 /126**: 表示最后 2 位是主机位（$128 - 126 = 2$）。
- **块大小 (Block Size)**: $2^2 = 4$。
- **计算范围**:
  - 基准地址是 `...::c` (十六进制 c = 十进制 12)。
  - 范围是 `12` 到 `15`（即 `c, d, e, f`）。
  - **注意**: 这里题目中的描述 `...::1f` 实际上对应的是子网内的一个具体 IP。
  - *校注：在 Cisco 的某些题库版本中，地址位的偏移计算较为诡异，最稳妥的方法是优先死记端口号对应关系，因为端口号在 ACL 语句末尾是唯一的。*

### **快速记忆技巧**

在考场上，如果 IPv6 计算让你头晕，直接看 `eq` 后面的数字：

1. 看到 **NTP** $\to$ 找末尾 **123** 的方块。
2. 看到 **syslog** $\to$ 找末尾 **514** 的方块。
3. 看到 **HTTP** $\to$ 找末尾 **80** 的方块。
4. 看到 **HTTPS** $\to$ 找末尾 **443** 的方块。

这样即使不看复杂的 IPv6 地址，也能 100% 做对

| **左侧 ACL 语句 (Source Subnet & Port)** | **右侧描述 (Purpose & Specific Host)**      |
| ---------------------------------------- | ------------------------------------------- |
| **permit ip ...::c/126 ... eq 123**      | **Permit NTP from this source ...::1f**     |
| **permit ip ...::e/126 ... eq 514**      | **Permit syslog from this source ...::1c**  |
| **permit ip ...::800/117 ... eq 80**     | **Permit HTTP from this source ...::0fff**  |
| **permit ip ...::/117 ... eq 443**       | **Permit HTTPS from this source ...::07ff** |
