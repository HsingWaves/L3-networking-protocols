LDP 会话的建立过程可以分为两个主要阶段：**发现阶段（Discovery）** 和 **会话建立阶段（Session Establishment）**。

#### 1. 发现阶段 (UDP 646)

邻居之间通过发送 LDP Hello 报文（使用 UDP 端口 646）来互相发现。在这些 Hello 报文中，包含了一个重要的字段：**LDP Identifier (LSR ID)**。

#### 2. 会话建立阶段 (TCP 646)

一旦两个路由器互相发现，它们就需要建立一个 **TCP 连接** 来交换标签。

- **谁来发起连接？** 为了避免两个路由器同时发起 TCP 连接导致资源浪费，LDP 协议规定了一个“主从”规则。
- **判定标准：** 比较双方的 **传输地址 (Transport Address)**，通常就是 LSR ID（也就是 IP 地址）。
- **判定结果：** **拥有更高 (Higher) IP 地址** 的路由器将扮演 **主动角色 (Active Role)**，负责发起 TCP 三次握手。IP 地址较低的路由器则扮演 **被动角色 (Passive Role)**，等待连接。

------

### 知识点扩展

- **LDP ID 的结构：** 格式通常为 `<IP_Address>:<Label_Space_ID>`（例如 `1.1.1.1:0`）。
- **为什么是 Higher？** 这是一种协议约定的硬性规定，类似于 BGP 中 Router ID 较高的一方通常在建立连接时具有特定的行为特征，旨在确保连接建立过程的唯一性和确定性。
- **TCP 端口：** LDP 的 TCP 连接使用的是端口 **646**。
