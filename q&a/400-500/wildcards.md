### ACL 通配符掩码 (Wildcard Bits)

- **核心点**：通配符位为 `0` 表示必须匹配，为 `1` 表示可以忽略。
- **口算技巧**：
  - `0.0.3.255` 意味着第三字节可以有 $2^2=4$ 个变化。
  - 起始位如果是 `4`（二进制 `...00100`），那么覆盖范围就是 `4, 5, 6, 7`。
  - 起始位如果是 `12`（二进制 `...01100`），那么覆盖范围就是 `12, 13, 14, 15`。

在 CCNP 考试或实际网络排障中，快速口算**通配符掩码（Wildcard Mask）\**的诀窍在于\**“块大小（Block Size）”**的概念，而不是去硬啃二进制。

以下是三步口算法：

------

### 第一步：识别“块大小” (Block Size)

通配符掩码的第三或第四字节通常是 $2^n - 1$。看到这些数字，直接反应出它代表的“块大小”：

| **通配符位**   | **块大小 (Block Size)** | **含义**                      |
| -------------- | ----------------------- | ----------------------------- |
| **0.0.0.255**  | **256**                 | 匹配 1 个完整的 C 类网段      |
| **0.0.3.255**  | **4**                   | 匹配 **4** 个连续的 C 类网段  |
| **0.0.7.255**  | **8**                   | 匹配 **8** 个连续的 C 类网段  |
| **0.0.15.255** | **16**                  | 匹配 **16** 个连续的 C 类网段 |

------

### 第二步：检查“起始位”是否是块大小的倍数

这是最关键的一步。要使 ACL 精确匹配，**起始 IP 的对应字节必须是块大小的整数倍**。

- **例子 A（你的题目）**：`192.168.4.0` 搭配 `0.0.3.255`。
  - 块大小是 **4**。
  - 起始位 **4** 是 4 的倍数吗？**是**。
  - **范围计算**：$4$ 到 $4 + 3 = 7$。所以匹配的是 `.4, .5, .6, .7` 这四个网段。
- **例子 B**：`192.168.12.0` 搭配 `0.0.3.255`。
  - 块大小是 **4**。
  - 起始位 **12** 是 4 的倍数吗？**是** ($4 \times 3$)。
  - **范围计算**：$12$ 到 $12 + 3 = 15$。所以匹配的是 `.12, .13, .14, .15`。

------

### 第三步：万能公式（推算范围）

如果你拿到一个 IP 和掩码，想看它到底覆盖到哪，用这个公式：

> **起始地址 + 通配符位 = 结束地址**

**实战演练：**

如果题目给你 `172.16.16.0 0.0.15.255`，范围是多少？

1. 第三字节起始是 `16`。
2. 通配符位是 `15`。
3. $16 + 15 = 31$。
4. **结论**：该 ACL 覆盖了 `172.16.16.0` 到 `172.16.31.255` 的所有地址。

------

### 

1. **非倍数陷阱**：如果题目给 `192.168.5.0 0.0.3.255`。
   - 虽然 $5+3=8$，但因为 **5 不是 4 的倍数**，这个 ACL 在路由器底层会被拆解或产生歧义，它**不会**精准匹配 5, 6, 7, 8。它会从最近的倍数 4 开始匹配。
2. **全 1 变全 0**：记住掩码和通配符的互换公式：$255.255.255.255 - 子网掩码 = 通配符掩码$。
   - 比如 `/22` 掩码是 `255.255.252.0`。
   - $255 - 252 = 3$。
   - 所以 `/22` 对应的通配符就是 `0.0.3.255`。

虽然技术上可行，但在生产环境和考试中，我们几乎只用 $2^n - 1$：

1. **可读性极差**：后来的工程师看到 `0.0.0.254` 可能会怀疑人生。
2. **硬件加速限制**：许多旧款交换机的硬件芯片（TCAM）专门针对连续掩码进行了优化。使用非 $2^n - 1$ 的掩码可能会导致匹配效率下降，甚至被迫交给 CPU 处理（导致丢包或延迟）。
3. **协议不支持**：OSPF 的 `network` 命令在某些旧版本中，如果检测到非连续掩码，会直接报错。

------

### 

- **在考试里**：请死死认定通配符掩码必须是 $2^n - 1$（如 0, 1, 3, 7, 15, 31, 63, 127, 255）。
- **在实际工作中**：如果你想玩出花来（比如只匹配某个范围内的奇数或特定位），你可以利用非 $2^n - 1$ 的特性，但一定要写好文档注释。
