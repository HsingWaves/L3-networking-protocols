# 分析执行流程

## 情况 1：匹配 ACL NON-CISCO

命中 sequence 5（deny）

结果：

- 不执行 set
- 直接回到正常路由查表
- 不再往下匹配

👉 所以这些流量 **不会被 sequence 10 评估**

------

## 情况 2：不匹配 ACL

sequence 5 不匹配 → 继续往下

进入 sequence 10（permit）

sequence 10 没 match = match any

执行：

```
set ip next-hop 192.168.1.5
```

👉 所有其他流量都被改下一跳

------

# 所以两个正确效果是：

### ✅ A. Packets are forwarded using normal route lookup.

（匹配 ACL 的流量走正常路由）

### ✅ D. Packets are evaluated by sequence 10.

（没匹配 ACL 的流量会被 seq 10 处理）
