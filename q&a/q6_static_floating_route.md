你配置的是：

```
ip route 192.168.2.2 255.255.255.255 209.165.200.225 130
```

这里最后的 **130** 不是 metric，而是 **Administrative Distance（管理距离，AD）**。

而路由表里你看到的是：

```
O 192.168.2.2 [110/11] via 192.168.12.2, Ethernet0/0
```

OSPF 路由的 **AD = 110**（这是默认值）。

静态路由配置成了“浮动静态路由 (floating static route)”——它的管理距离 AD 比 OSPF 更大，所以路由表不会选它

## 为什么会选 OSPF 的下一跳，而不是你写的 209.165.200.225？

路由选择顺序是：

1. **先比 AD（管理距离）**：越小越可信
2. 如果 AD 一样，才会比 **metric（度量值）**（比如 OSPF cost）

现在：

- 静态路由 AD = **130**
- OSPF 路由 AD = **110**

因为 **110 < 130**，所以路由器认为 **OSPF 更可信**，于是把到 `192.168.2.2/32` 的路由装进 RIB/FIB 的是 **OSPF 那条**，下一跳自然变成 `192.168.12.2`。

------

## 对应选项

✅ **C. The configured AD for the static route is higher than the AD of OSPF.**

------

## 补一句：为什么有人要把静态路由 AD 设大？

这就是 **floating static route** 的典型用法：

- 正常情况下用动态路由（OSPF）
- 动态路由挂了（OSPF 路由消失）时，才“浮上来”用静态路由做备份

如果你希望静态路由优先，就把 AD 设得比 110 小，或者干脆不写 AD（默认静态 AD=1）。
