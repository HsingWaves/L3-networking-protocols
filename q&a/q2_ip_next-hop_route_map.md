# 🔵 第一部分：/30 和 /31 到底是什么？

## 1️⃣ `/数字` 是什么？

```
/数字 = 前缀长度
```

意思是：

> 前多少位是网络位（IPv4 一共 32 位）

例如：

| 前缀 | 网络位 | 主机位 |
| ---- | ------ | ------ |
| /30  | 30 位  | 2 位   |
| /31  | 31 位  | 1 位   |

------

# 🔵 第二部分：/30 是什么？

```
/30 = 255.255.255.252
```

主机位：

```
32 - 30 = 2 位
```

2 位能表示：

```
00
01
10
11
```

= 4 个地址

但：

| 地址 | 作用 |
| ---- | ---- |
| .0   | 网络 |
| .1   | 主机 |
| .2   | 主机 |
| .3   | 广播 |

👉 可用主机 = 2 个

------

## 📌 用途

传统点到点链路。

------

# 🔵 第三部分：/31 是什么？

```
/31 = 255.255.255.254
```

主机位：

```
32 - 31 = 1 位
```

只能表示：

```
0
1
```

理论上：

- 1 网络
- 1 广播

但 RFC 3021 规定：

👉 点到点链路不需要广播

所以：

两个都可用。

------

## 📌 结论

| 子网 | 可用主机 | 广播 |
| ---- | -------- | ---- |
| /30  | 2        | 有   |
| /31  | 2        | 没有 |

------

# 🔵 第四部分：心算主机数（3 秒法）

核心公式：

```
可用主机 = 2^(32 - 前缀) - 2
```

（/31 特例不减 2）

# 🔵 第五部分：结合刚才那题

```
172.20.40.0/30
```

- 2 位主机位
- 只有 .1 和 .2 可用

图中写：

```
Gi0/2  .2
```

说明：

```
R1 = 172.20.40.2
```

那对端只能是：

```
R2 = 172.20.40.1
```

所以：

```
set ip next-hop 172.20.40.1
```

# 🔵 第六部分：真正理解点到点逻辑

在 /30 或 /31 点到点链路上：

只有两台设备。

如果你看到：

```
本地 = .2
```

那对端一定是：

```
.1
```

这不是猜，是逻辑排除。

# 🔵 最终核心总结

```
/30 = 传统P2P，有广播
/31 = 现代P2P，无广播

/数字 = 网络位
32 - /数字 = 主机位
```

# 🚀 怎么 3 秒记住？

看规律就行：

主机位每少 1 位：

```
地址数量 ÷ 2
```

例如：

```
/24 = 256
/25 = 128
/26 = 64
/27 = 32
/28 = 16
/29 = 8
/30 = 4
/31 = 2
```

这是 2 的幂次方：

```
2^8 2^7 2^6 2^5 2^4 2^3 2^2 2^1
```
