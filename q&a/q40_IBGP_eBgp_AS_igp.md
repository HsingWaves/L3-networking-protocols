非常好的问题 👍
这已经是“设计哲学层面”的问题了，而不是死记数字。

你问的是：

> 为什么 iBGP AD=200？
> 是因为它是内部的吗？

答案是：

❌ 不是因为“内部”
✅ 是因为 **Cisco 认为 IGP 比 iBGP 更值得信任**

------

# 1️⃣ 先搞清：AD 是什么？

AD = Administrative Distance
👉 路由来源的“可信度排序”
👉 数字越小越优先

注意：

> AD 只在本路由器内部比较
> 不是协议的全局属性

------

# 2️⃣ Cisco 默认 AD 表

| 协议  | AD   |
| ----- | ---- |
| 直连  | 0    |
| 静态  | 1    |
| eBGP  | 20   |
| EIGRP | 90   |
| OSPF  | 110  |
| iBGP  | 200  |

------

# 3️⃣ 为什么 iBGP 这么低优先级（200）？

核心原因：

## 🔹 iBGP 不是“收敛协议”

IGP（OSPF/EIGRP）设计目标是：

- 快速收敛
- 精确计算最短路径
- 描述拓扑结构

而 BGP 设计目标是：

- 策略控制
- 跨 AS 传播
- 不关心内部最短路径

------

### 举个工程现实例子

假设：

AS 内部有 OSPF
同时你通过 iBGP 学到一个前缀

如果 iBGP AD 比 OSPF 小：

那内部流量可能绕去边界路由器
而不是走本地最短路径

这会：

- 流量不对称
- 收敛慢
- 黑洞

所以 Cisco 设计理念是：

> 内部 IGP 永远优先
> iBGP 只是传播“外部前缀”

------

# 4️⃣ 关键本质

## 🔥 IGP = 描述网络结构

## 🔥 iBGP = 传播外部可达性

所以：

IGP 更可信
iBGP 只是策略载体

因此给它 AD=200。

------

# 5️⃣ 为什么 eBGP 却是 20？

因为：

eBGP 是“真正外部来源”

当你和 ISP 建邻居：

你肯定希望优先用 ISP 的路由
而不是内部猜出来的。

所以：

```text
eBGP = 20 （非常优先）
iBGP = 200 （最低）
```

------

# 6️⃣ 回到你那题

R3 同时看到：

- IGP 版本（来自重分发）
- iBGP 版本（来自 R4）

因为 iBGP AD=200
被 IGP（110）压过去

于是出现回灌。

------

# 7️⃣ 所以“内部”不是原因

很多人误解为：

i = internal
所以 AD 低

不是。

真正逻辑是：

> iBGP 不能破坏内部 IGP 的拓扑选择

这是设计原则。

------

# 8️⃣ 一句话总结

```text
IGP = 拓扑真相
iBGP = 外部消息传播

为了不让外部消息破坏内部拓扑，
Cisco 把 iBGP AD 设成 200。
```

------

好，这个要一次讲透 —— **AS 是“地盘”概念，IGP 是“地盘内部怎么走路”**。

你现在在做 CCNP/重分发题，这个必须绝对清晰。

# 1️⃣ AS 是什么？

## 🔹 AS = Autonomous System（自治系统）

👉 一个统一管理的网络
👉 有一个唯一编号（ASN）
👉 对外是一个整体

例如：

- AS 64512
- AS 65001
- AS 13335（Cloudflare）
- AS 15169（Google）

你可以理解为：

> AS = 一个国家
> 内部怎么修路自己决定
> 对外统一对外交往

------

# 2️⃣ IGP 是什么？

## 🔹 IGP = Interior Gateway Protocol

👉 AS 内部使用的路由协议
👉 负责“内部怎么走”

常见 IGP：

- OSPF
- EIGRP
- IS-IS
- RIP（老）

------

# 3️⃣ AS 和 IGP 的关系图

```
          AS 64512
   -----------------------
   |   OSPF / EIGRP     |
   |   内部怎么走路     |
   -----------------------
```

AS 内部用 IGP。

------

# 4️⃣ BGP 和 AS 的关系

BGP 是：

> AS 和 AS 之间的协议

```
AS 64512 ---- AS 65001
      eBGP
```

------

# 5️⃣ 完整对比表

| 名称 | 作用范围            | 举例         |
| ---- | ------------------- | ------------ |
| AS   | 一个自治网络整体    | AS64512      |
| IGP  | AS 内部路由         | OSPF / EIGRP |
| eBGP | AS 之间             | ISP 互联     |
| iBGP | AS 内部传播外部路由 | 数据中心核心 |

------

# 一句话核心区别

| 类型     | 全称                    | 在哪种情况用   | 默认 AD |
| -------- | ----------------------- | -------------- | ------- |
| **BGP**  | Border Gateway Protocol | 协议本身       | —       |
| **eBGP** | external BGP            | 不同 AS 之间   | **20**  |
| **iBGP** | internal BGP            | 同一个 AS 内部 | **200** |

------

# 1️⃣ BGP 是什么？

**BGP = 路由协议本身**

它是：

- 互联网核心路由协议
- Path Vector 协议
- 通过 AS-PATH 防环

简单说：

> OSPF/EIGRP 负责公司内部
> BGP 负责公司之间

------

# 2️⃣ eBGP 是什么？

**eBGP = 不同 AS 之间**

比如：

```
AS 65001 ---- AS 64512
   R3         R4
```

这就是 eBGP。

特点：

- 默认 AD = **20（非常优先）**
- TTL 默认 1（直连）
- 会修改 next-hop
- 会增加 AS-PATH

------

# 3️⃣ iBGP 是什么？

**iBGP = 同一个 AS 内部**

比如：

```
AS 64512

R1 ----- R2 ----- R3
```

都在 AS 64512 内部

特点：

- 默认 AD = **200（很低优先级）**
- TTL 默认 255
- 不修改 AS-PATH
- 不自动传递给其他 iBGP（需要 RR）

------

# 4️⃣ 最关键差异图

## 🔥 eBGP

```
AS 100 ---- AS 200
   100 200
```

AS-PATH 会变成：

```
200 100
```

------

## 🔥 iBGP

```
AS 200 内部
R1 → R2
```

AS-PATH 不变。

------

# 5️⃣ 为什么 iBGP AD=200？

因为 Cisco 认为：

> 内部 IGP（OSPF/EIGRP）更可信
> BGP 是“外部来的”

所以：

```
eBGP = 20
EIGRP = 90
OSPF = 110
iBGP = 200
```

这就是你那题抖动的根本原因。

------

# 6️⃣ 实际工程理解

## eBGP

👉 ISP 之间
👉 企业和 ISP 之间

------

## iBGP

👉 数据中心核心
👉 双核心路由器
👉 路由反射

------

> iBGP 用于 AS 内部，AD 200
> iBGP 默认不改 next-hop，不自动转发给其他 iBGP

# 🚦 三、IGP vs BGP 本质区别

| 对比点       | IGP      | BGP     |
| ------------ | -------- | ------- |
| 使用范围     | AS 内部  | AS 之间 |

# 🔁 五、BGP 传播规则总图

- TTL=255

# 🔄 六、重分发危险模型（高频坑）

```
BGP (AD=200)
     ↓
Redistribute
     ↓
IGP (AD=110)
     ↓
回到原路由器
     ↑
IGP 更优，被选
```

产生：

- 回灌
- 抖动
- 非对称

解决：

```
distance bgp 20 100 200
```

让 iBGP=100 < OSPF=110
