### 

**IPv6 Destination Guard** 的核心作用是过滤发往“未解析”地址的流量，防止扫描攻击导致路由器资源耗尽。

- **原理**：在 IPv6 中，当路由器收到去往某个子网地址的包，但该地址在邻居缓存（Neighbor Cache）中不存在时，路由器会发起 **Neighbor Solicitation (NS)** 来解析地址。
- **攻击场景**：攻击者可以发送大量伪造的目的 IP 地址（扫描一个巨大的 `/64` 网段）。
- **结果**：如果没有保护，路由器会为每个伪造地址生成待处理的邻居表项并发送 NS 报文。这会迅速撑爆路由器的内存（邻居表上限）并消耗 CPU。
- **防御**：**Destination Guard** 会检查目的地址是否在“绑定表（Binding Table）”中。如果该地址从未被合法分配或发现，流量会直接被丢弃，从而保护路由器不被 **DoS** 攻击拖垮。

------

### 其他选项的功能区分

在考试中，这几个概念非常容易混淆，我们可以通过它们的“防御对象”来区分：

| **选项**              | **防御的对象 (What it stops)**                  | **类似 IPv4 的功能**          |
| --------------------- | ----------------------------------------------- | ----------------------------- |
| **RA Guard**          | 伪造的路由器（防止非法 RA 报文）。              | 类似于关闭了无用端口的接入。  |
| **DHCPv6 Guard**      | 非法的 DHCPv6 服务器（防止分配错误的 IP/DNS）。 | **DHCP Snooping**             |
| **Destination Guard** | **扫描/泛洪攻击 (DoS)**。                       | N/A (IPv6 特有，因为网段太大) |
| **MAC filtering**     | 特定主机的访问控制。                            | MAC 地址过滤                  |

------

### 总结

题目问的是 **"minimize denial of service attacks"**，而扫描 `/64` 网段导致路由器内存溢出是 IPv6 最典型的本地 DoS 方式。因此，**Destination Guard** 是唯一针对此场景设计的组件。

你现在正在复习 IPv6 安全，需要我帮你理一下 **SAVI (Source Address Validation Improvement)** 的逻辑吗？它是将 RA Guard、DHCPv6 Guard 等结合在一起的终极防御架构。
