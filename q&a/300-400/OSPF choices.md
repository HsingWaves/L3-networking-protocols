想要干预 OSPF 选路时，顺序通常是：

1. **第一优先级：类型切换**（把 E2 改成 E1，让路由器考虑内部开销）。
2. **第二优先级：修改 Cost**（在接口上 `ip ospf cost` 或者重发布时指定 `metric`）。
3. **最后手段：修改 AD**（通常只在解决多点双向重发布导致的路由回馈/环路时才用）。

**一句话总结：** Type 1 是让路由器**变聪明**（学会算路），而改 AD 是给路由器**下死命令**（强行不信某人）。在网络工程里，我们总是倾向于让协议自己算出最优路径。



修改 OSPF Cost（度量值）主要有两种方式，具体取决于你是想影响**普通区域内/间路由**，还是影响**重发布进来的外部路由**。

以下是具体的配置命令：

### 1. 接口级修改：`ip ospf cost`

这是最常用的方法，直接改变经过某个物理接口的开销。它会影响**所有类型**的 OSPF 路由（O, O IA, O E1, O E2）。

- **配置命令：**

  Bash

  ```
  R1(config)# interface GigabitEthernet0/1
  R1(config-if)# ip ospf cost 50  # 将此接口的开销手动改为 50
  ```

- **应用场景：** 当你有两条路径，想让流量避开某条较慢的链路时。

------

### 2. 重发布时修改：`metric`

当你把其他协议（如 EIGRP, BGP 或直连路由）引入 OSPF 时，可以手动指定初始开销。

- **配置命令：**

  Bash

  ```
  R1(config)# router ospf 1
  R1(config-router)# redistribute eigrp 100 metric 500 subnets
  ```

  *注意：如果不指定，OSPF 重发布默认的 Metric 是 **20**（重发布 BGP 除外，BGP 默认是 1）。*

- **应用场景：** 你有两个 ASBR，想让其中一个 ASBR 通告的路由看起来“代价更高”，从而让全网优选另一个 ASBR。

------

### 3. 修改外部路由类型：`metric-type`

这就是你刚才做的那道题涉及的命令。

- **配置命令：**

  Bash

  ```
  R1(config-router)# redistribute eigrp 100 metric-type 1 subnets
  ```

  - **type 1**: 对应 **O E1**。
  - **type 2**: 对应 **O E2**（OSPF 的默认行为）。

------

### 4. 全局修改参考带宽：`auto-cost reference-bandwidth`

OSPF 的默认计算公式是 $Cost = \frac{100 \text{ Mbps}}{\text{接口带宽}}$。在现在动辄万兆（10Gbps）的环境下，100M、1G 和 10G 的 Cost 算出来都是 1，这会导致选路错误。

- **配置命令：**

  Bash

  ```
  R1(config-router)# auto-cost reference-bandwidth 100000 
  # 将参考带宽改为 100Gbps，这样万兆和千兆接口就能区分出 Cost 了
  ```

  *注意：这个命令必须在网络中**所有**路由器上同步修改，否则选路会乱套。*

------

### 总结

| **修改对象**         | **命令**                          | **影响范围**                  |
| -------------------- | --------------------------------- | ----------------------------- |
| **特定链路**         | `ip ospf cost <value>`            | 经过该接口的所有流量          |
| **外部路由初始值**   | `redistribute ... metric <value>` | 仅限该 ASBR 注入的外部路由    |
| **外部路由计算逻辑** | `redistribute ... metric-type 1`  | 强制全网计算内部链路开销 (E1) |



你的理解方向很对，但在网络协议的“地位等级”中，**顺序正好反过来了**。

我们要区分两个概念：一个是**协议内部**的选路，一个是**不同协议之间**的选路。

### 1. 宏观地位：AD > Type > Metric

当路由器收到多条去往同一个目的地的路由时，它会按以下顺序进行筛选（从高到低）：

1. **AD (Administrative Distance - 管理距离)**：
   - **地位：最高。**
   - **作用：** 决定“听谁的”。如果 EIGRP（AD 90）和 OSPF（AD 110）都告诉你怎么去迪拜，路由器**直接选 EIGRP**，根本不看后面的参数。
2. **Type (Route Type - 路由类型)**：
   - **地位：中等（仅在协议内部）。**
   - **作用：** 如果大家都是 OSPF，那么“区域内路由 (O)” 永远优于 “外部路由 (E1/E2)”，即使外部路由的 Metric 更小。
3. **Metric (Cost / Bandwidth - 度量值)**：
   - **地位：最低（最后的决胜局）。**
   - **作用：** 只有当 **协议一样** 且 **类型也一样** 时，才会比拼 Metric。这时候你改的 `bandwidth` 才会起作用。

------

### 2. EIGRP 的特殊性

在你给出的 EIGRP 题目中，三台路由器都运行同一种协议（EIGRP 222），且都是内部路由。所以：

- **AD 相同**（都是 90）。
- **Type 相同**（都是内部路由）。
- **唯一比拼的就是 Metric**。

这就是为什么题目让你改 `bandwidth`。因为在 AD 和 Type 都拉不开差距的情况下，**Metric 就是唯一的选路标准**。

------

### 3. EIGRP Metric 的“级别”

如果你是指 EIGRP Metric 计算公式内部的权重（K值），那么 **Bandwidth（带宽）** 和 **Delay（延迟）** 是默认开启的两个核心指标：

- **Bandwidth (K1)**：它是路径上所有链路中**最小值**（取最小值）。
- **Delay (K3)**：它是路径上所有链路的**累加值**（取总和）。

> **顺口溜总结：**
>
> **不同协议比 AD，**
>
> **同种协议比 Type，**
>
> **同种类型比 Metric。**

------

### 一个极端的例子帮你记住：

假设你有两条路：

- **路径 A**：EIGRP 协议，带宽只有可怜的 64 Kbps。
- **路径 B**：OSPF 协议，带宽是豪横的 10 Gbps。

**路由器会选哪条？**

答案是 **路径 A**。因为 EIGRP 的 AD (90) 比 OSPF 的 AD (110) 更小（更可信）。即使 A 路慢得像蜗牛，路由器也会因为它“出身好”而选择它。这就是 **AD > Metric** 的力量。





| **路由类型**   | **对应 LSA** | **汇总命令**      | **配置位置** |
| -------------- | ------------ | ----------------- | ------------ |
| **区域间路由** | Type 3       | `area [ID] range` | **ABR**      |
| **外部路由**   | Type 5 / 7   | `summary-address` | **ASBR**     |

> **面试/考试小贴士：**
>
> 如果你在 ABR 上看到大量的外部路由（E1/E2），你不能在 ABR 上通过 `summary-address` 缩小路由表，你只能回到源头——即产生这些路由的 **ASBR** 上去操作。
