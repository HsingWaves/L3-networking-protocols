需要把“External”这个词拆解成两个层面来看，因为 BGP 和 IGP（如 OSPF/EIGRP）对它的定义略有不同。

------

## 1. BGP 层面：EBGP vs. IBGP

这是最直接的区分方式，看的是 **邻居关系 (Peering)**：

- **EBGP (External BGP)：** 发生在**不同 AS** 之间（比如 R3 AS 64512 和 R4 AS 1234）。
  - **管理距离 (AD)：** 20（非常可信）。
- **IBGP (Internal BGP)：** 发生在**同一个 AS** 内部。
  - **管理距离 (AD)：** 200（不怎么可信，为了防止环路）。

------

## 2. 路由属性层面：External 路由

这是看路由的**来源**。在重发布（Redistribution）的场景下，“External”指的是“不属于本协议原生定义的路由”。

回看你发的那张图，这就是最混乱的地方：

- **对 EIGRP 来说：** 凡是从 BGP 或 OSPF 重发布进来的，都叫 **EIGRP External** 路由。
  - **AD 值：** **170**（普通的 EIGRP 内部路由是 90）。
- **对 OSPF 来说：** 凡是从 EIGRP 重发布进来的，都叫 **OSPF External** (LSA Type 5)。
  - **AD 值：** **110**（OSPF 比较特殊，内部外部默认都是 110）。

------

## 3. 为什么你那张图会产生“回灌”？

就是因为这些 **External AD 值** 在打架。我们用“AD 竞争”的视角重看一遍：

1. **R3 视角：** 它从 R4 学到 BGP 路由（AD 20）。这是它的“真爱”，它会把这跳路由放入路由表。
2. **R1 视角：** 它从 EIGRP 学到这条路由。因为是从 R3 重发布进来的，R1 看到的 AD 是 **170**。
3. **重发布动作：** R1 把这跳 AD 170 的路由注入 OSPF。
4. **致命的回灌：** R3 此时从 OSPF 收到这条路由，AD 是 **110**。

**这就是陷阱：**

- 原本路由是：`R4 (BGP 20) -> R3 -> R1 (EIGRP 170)`
- 回灌后变成：`R1 -> R3 (OSPF 110)`
- 如果 R3 原始的 BGP 链路挂了，R3 会觉得：“嘿，虽然 BGP 没了，但 OSPF 传给我的这条路 AD 只要 110，比 EIGRP 的 170 还优！”
- 于是 R3 就会把流量扔给 R1，R1 又扔给 R2，R2 扔回 R3…… **环路达成**。

------

## 总结

- **不同 AS 之间**建立的 BGP 叫 **EBGP**。
- **不同协议之间**交换的路由叫 **External Route**。
- **核心痛点：** 不同的协议对“外部路由”的信任程度（AD 值）不一样。BGP 极度信任外部 (20)，而 EIGRP 极度怀疑外部 (170)。

****

**只要是重发布（Redistribute），无论源头和目的地是否在同一个 AS 内，在接收协议看来，它永远算“External（外部）”。**

“Internal”和“External”的区别，不在于地理位置或 AS 编号，而在于**“路由协议的血统”**。

------

### 1. 为什么“同一 AS”重发布也算 External？

假设你在 AS 100 内部同时运行了 OSPF 和 EIGRP。

当你把 OSPF 的路由重发布进 EIGRP 时：

- **数据结构变了：** OSPF 的路由携带的是 Cost（开销），而 EIGRP 使用的是五元组度量值（带宽、延迟等）。
- **信息丢失：** 在转换过程中，原协议特有的细节（比如 OSPF 的区域 ID、LSA 类型）全部丢失了。
- **不可靠性：** 对于 EIGRP 进程来说，这条路由不是它“亲手”算出来的，而是别人“告诉”它的。为了安全，它必须给这条路由贴上 **External (D EX)** 的标签，并分配更高的 AD 值（170）。

------

### 2. 深度对比：原生 vs. 重发布

| **特性**       | **原生宣告 (Network)**    | **重发布 (Redistribute)**                 |
| -------------- | ------------------------- | ----------------------------------------- |
| **术语**       | Internal (内部)           | External (外部)                           |
| **血统**       | 纯正 (原生协议发现并计算) | 杂交 (从其他协议转换而来)                 |
| **EIGRP 表现** | 标记为 `D`，AD = 90       | 标记为 `D EX`，AD = 170                   |
| **OSPF 表现**  | LSA Type 1/2/3            | **LSA Type 5 (External)**                 |
| **防环机制**   | 强（利用协议自身的算法）  | **弱**（容易产生次优路径，需靠 Tag 防护） |

------

### 3. 一个极端的例子：BGP 内部重发布

你在同一台路由器上，把 **IBGP**（同一 AS）重发布进 **OSPF**（同一 AS）：

- 虽然 AS 号一样，但 OSPF 依然会把它当成 **Type 5 LSA (External)**。
- 它的 AD 值依然是 110。
- 它依然需要手动配置 `subnets` 关键字（对于老版本 IOS），否则 OSPF 甚至不屑于携带这种外部路由。

------

### 4. 为什么这个结论对 CCNP 考生很重要？

在做“双向重发布”实验时，如果你潜意识里认为“反正都在一个 AS 里，应该算内部路由吧”，你就会忽略 **AD 值的差异**。

回到你之前那张图：

1. 如果 R1 误以为从 EIGRP 重发布进来的路由也是“内部”的。
2. 它可能就不会意识到这跳路由的 AD 值在不同协议间跳变（170 -> 110 -> 20）。
3. 正是因为它变成了 **External**，它才携带了特殊的“外部属性位”，让我们能给它打上 **Tag**。

**总结一句话：**

在路由器的眼里，**Network 是“自家人”，Redistribute 是“外来务工人员”**。哪怕外来务工人员来自同一个村（AS），在协议的户口本上，它依然属于“外部人员”。
